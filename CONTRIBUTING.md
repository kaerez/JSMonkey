## 2. Structure of This Repository

Because this repository hosts multiple standalone applications, pay close attention to directory-level licenses:
* **Global License:** Unless explicitly specified otherwise within a sub-folder, the overall repository falls under the **GNU AGPLv3**.
* **Sub-Project Licenses:** Check for a localized `LICENSE` file within individual sub-directories before writing code. Your contributions to those directories will automatically adopt that specific sub-license.

---

## 3. Contribution Workflow

To submit your code for review, please follow these steps:

1. **Fork the Repository:** Create your own personal fork of the project.
2. **Create a Feature Branch:** Isolate your changes in a descriptive branch (e.g., `feature/login-fix`).
3. **Keep Commits Clean:** Write brief, clear commit messages detailing *what* changed and *why*.
4. **Submit a Pull Request (PR):** Open a PR against our `main` or `development` branch. 

*Note: Pull requests containing code copied from other closed-source or inappropriately licensed open-source projects will be rejected immediately.*
